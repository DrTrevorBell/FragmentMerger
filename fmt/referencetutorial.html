Fragment Merger Tool


Reference Tutorial One or more reference sequences (in FASTA format) may be specified on the input page. These sequences are not used to create the merge. After the merge has been created, a multiple sequence alignment of the merged sequence with the reference sequence/s is performed, using the fast muscle program. The alignment is displayed on the output page. Loci which are conserved across all sequences (reference and merged) are indicated with a | character. Mismatches are indicated with a space character.

The alignment is inteded to be used a quick check to validate the success and/or accuracy of the merge. When fragments which constitute a full-length circular genome (such as a viral genome) are specified, it is likely that the final merged sequence will not be in the standard orientation. The same scenario will occur when sequence data straddling the "start" position of a circular genome if specified. In the former scenario, the merge sequence can be slid into the correct orientation by specifying appropriate "slide" parameters on the input page. In either scenario, however, specifying a single reference sequence in the standard orientation will result in an alignment with long regions of gaps on one or either end. This can be avoided by preparing the refernce sequence to match the orientation/position of the merged sequence, or by preparing a double-length reference sequence.

If the reference sequence file contains many sequences, more time will be required to generate the alignment. Typically, one or two reference sequences should be sufficient.
